## 目标
- 将日志配置集中到 `config.py`，一次性初始化后全局生效，移除各模块内的分散配置。
- 精简策略基类接口为两个公开方法：`check_stock`、`explain`；将预处理改为受保护方法 `_preprocess` 并在 `check_stock` 内部调用。
- 简化 `run_strategies.py`：只调用策略的两个公开方法，不再显式调用预处理与冗余调试逻辑。

## 日志配置统一
- 在 `config.py` 添加日志初始化：
  - `setup_logger(level_console='INFO', level_file='DEBUG', file_pattern='logs/run_%Y%m%d_%H%M%S.log')`
  - 负责：创建 `logs/` 目录、移除默认 handlers、添加终端 INFO、文件 DEBUG 输出（UTF-8）。
- 应用方式：
  - 在入口脚本（`scripts/run_strategies.py`, `scripts/run_fetcher.py`）冒头 `from config import setup_logger; setup_logger()`。
  - 移除脚本内的 `logger.add` 等本地配置行。
- 可选配置：读取环境变量或 `config.py` 常量控制输出级别与文件命名。

## 精简策略基类
- 修改 `BaseStrategy`：
  - 将 `preprocess` 改为受保护抽象方法 `_preprocess(raw_df) -> DataFrame`。
  - `check_stock(ts_code, raw_df)` 内部首行调用 `df = self._preprocess(raw_df)`；具体策略在 `_preprocess` 中完成列映射与指标计算。
  - 保持 `explain(ts_code, raw_df)` 用于生成展示数据（可内部再次 `_preprocess` 或假设与 `check_stock` 使用同一原始输入）。
- 更新现有策略：
  - `ShaofuStrategy`：将现有 `preprocess` 重命名为 `_preprocess`；`check_stock` 首行改为 `_preprocess` 调用。

## 简化 run_strategies.py
- 只保留必要流程：
  - 读取 `basic_info` 构建 `name_map`（后续也可构建 `industry_map`、`exchange_map`）
  - 并发：每只股票执行 `check_stock`，命中则调用 `explain` 构建记录
  - 输出：打印汇总、保存 CSV/JSON
- 移除：
  - 本地 `logger.add` 配置（改为统一配置）
  - 显式调用 `strategy.preprocess` 的调试打印逻辑（不再依赖）

## 兼容与迁移
- 保持函数签名不变处：入口脚本的 CLI 参数不变（`--workers`、`--limit`、`--industries`）。
- 对外行为不变：终端 INFO 输出，文件 DEBUG 明细（由 config 控制）。
- 文档：在 README/注释中说明新的基类用法与统一日志配置使用方法。
